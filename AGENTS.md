# AGENTS.md

## General Rules

- Keep public documentation concise and safe.
- Do not commit real secrets, private URLs, personal data, or internal runbooks.
- Use placeholders in examples.
- Prefer small, reviewable changes.

## Public Repo Guidelines

- Replace names of real people with generic roles such as maintainer, developer, or team member.
- Replace production URLs with examples such as `https://your-api.example.com`.
- Keep architecture descriptions high-level unless the detail is intentionally public.
- Before publishing, run a local secret scan and inspect git history.
